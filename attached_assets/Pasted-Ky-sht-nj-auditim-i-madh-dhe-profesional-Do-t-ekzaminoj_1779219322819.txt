S-2 · Pa Rate Limiting — Vulnerable ndaj Brute Force dhe DoS
Risk: SERIOZ | Kategoria: Security

server.ts nuk ka asnjë express-rate-limit middleware. Çdo endpoint është i ekspozuar ndaj:

Brute force attack në login
DoS attack nëpërmjet flood-it të API calls
Abuse i POST /api/auth/2fa/challenge
Si rregullohet:

import rateLimit from 'express-rate-limit';
const apiLimiter = rateLimit({ windowMs: 15 * 60 * 1000, max: 100 });
const loginLimiter = rateLimit({ windowMs: 15 * 60 * 1000, max: 10 });
app.use('/api/', apiLimiter);
app.use('/api/auth/', loginLimiter);

S-3 · Internal Error Details Exposed ndaj Client
Risk: SERIOZ | Kategoria: Security

Shumë catch blocks kthejnë e.message direkt:

S-5 · Race Condition në Live Quiz — 100+ Studento Njëkohësisht
Risk: SERIOZ | Kategoria: Real-Time/Scalability

Endpoint POST /api/student/realtime-quiz/:sessionId/answer:

  return res.status(400)...

Dy requests që arrijnë pothuajse njëkohësisht mund të kalojnë check-un para se njëri të bëjë write (Node.js eshte single-threaded por await rqPersistSession krijon interrupt point). Me 100+ studento, mundësia e double-submission rritet.

Si rregullohet: Shto Set me "in-progress" tracking:

const processingAnswers = new Set<string>(); // sessionId:userId:qIndex

processingAnswers.add(key);
try { /* logic */ } finally { processingAnswers.delete(key); }

S-6 · File Upload — Pa Validim Tipi dhe Madhësie
Risk: SERIOZ | Kategoria: Security

  return res.status(400).json({ error: 'File too large.' });
